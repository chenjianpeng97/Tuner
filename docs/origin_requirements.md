# 记录一些原始需求想法

- 软件开发项目管理平台

**核心功能**:基于需求跟踪矩阵的软件开发项目管理工具

**设计构想**:
- 将需求跟踪矩阵表达成一个多维在线excel
- 多维在线表格可以以多种视图形态展示，如默认为需求，设计，测试这三列，称为项目跟踪视图（此视图可能包含客户需求，设计，测试，一级一些排期信息列）；项目跟踪视图下，在需求单元格可展示原始客户需求，但是可以操作这个需求单元格，跳转到查看这个客户需求相关的产品需求表中或新建产品需求表，这个产品需求表可灵活一些，表达传统开发的一级，二级，三级层级需求，或敏捷开发的epic,feature,user story等层级需求，这可以用自定义表头名称实现；关键的是从客户需求下钻进来，应维护一个客户需求和产品需求关联关系，在下钻过来的时候筛选出这个客户需求相关的产品需求进行维护，同时此时新增的产品需求自动维护到客户需求下；

**解决场景**:
- 任务需求不分家，通常小公司项目经理用客户会议上的一个excel表格记录一些功能开发需求，数据维护需求，运维需求等，以及这些需求的排期，预计开始/结束；这个平台需要让用户将这些需求和事项直接录入系统，然后区分哪些需求需要细化，以及他们关联的设计和测试信息；同时这个平台还需要提供一些视图，来展示这些需求的状态和进展情况
